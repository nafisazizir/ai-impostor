## Commands

```bash
pnpm dev          # Start dev server
pnpm build        # Production build
pnpm lint         # ESLint
pnpm test         # Run all tests (vitest)
pnpm vitest run lib/game/engine.test.ts  # Run a single test file
```

## What This Is

AI Impostor is an always-on spectator experience where 6 AI models (from different providers) play a Spyfall/Mr. White social deduction game continuously. Viewers watch in real time. Each game has 4 Civilians, 1 Impostor, and 1 Mr. White assigned across 6 fixed seats.

## Tech Stack

Next.js 16 (App Router) · React 19 · TypeScript 5 (strict) · Tailwind CSS v4 (`@tailwindcss/postcss`, NOT `tailwind.config.ts`) · shadcn/ui (base-nova style) · Vercel AI SDK (`generateObject()` + Zod schemas) · Vitest · pnpm

## Architecture

**No `src/` directory** — all source code lives at root level. Use `@/` import alias for all imports.

### Game Engine (`lib/game/engine/`)
Pure functional, event-sourced state machine. All transitions are `(GameState, Input) => GameState` with append-only events. Deterministic via injectable seeded RNG. Strict phase guards and turn-order enforcement.

Game flow: `setup → clue → discussion (multi-pass) → vote → elimination → [mr_white_guess] → [finished | next round]`

### AI Layer (`lib/ai/`)
- **`models.ts`** — 6 player models (one per seat) + 1 host model, all routed through AI Gateway with a single API key
- **`prompts.ts`** — Role-aware prompt builders with information hiding (civilians and impostor see identical prompt structure; Mr. White gets no word)
- **`actions.ts`** — AI action functions using `generateObject()` with Zod schemas and native reasoning capture
- **`runner.ts`** — Sequential game orchestrator driving the engine with AI decisions (10-round safety cap)

### Spectator UI (`components/game/`)
Live-only, connected to real-time SSE streaming via `useGameStream()` hook and `/api/game/stream` endpoint. Dark mode only (hardcoded on `<html>`).

### API Routes
- `GET /api/health/env` — Env validation health check
- `POST /api/game/run` — Runs a full AI game end-to-end (5-min max duration)

## Key Conventions

- Game terminology: "Seat" (1-6), "Player N (model-name)", roles are `civilian | impostor | mr_white`
- Tests live alongside source: `lib/**/*.test.ts`
- Tailwind v4 uses `@tailwindcss/postcss` — theme variables use oklch color values in `app/globals.css`
- shadcn/ui components live in `components/ui/`
- `next.config.ts` wraps config with `withWorkflow()` (Vercel Workflow DevKit)
- Fonts: Geist Sans + Geist Mono via `next/font/google`, exposed as CSS variables

## Environment Variables

Required: `AI_GATEWAY_API_KEY`, `UPSTASH_REDIS_REST_URL`, `UPSTASH_REDIS_REST_TOKEN`, `WORKFLOW_START_SECRET`
Optional: `WORKFLOW_LOOP_ID`, `WORKFLOW_MAX_DISCUSSION_PASSES`

Validated at runtime by `lib/config/env.ts` — missing keys fail fast.

## Development Status

See `DEV_NOTES.md` for the full delivery plan. Phases 0-1 (contracts + engine), Phase 3 (AI layer), and Phase 4 (spectator UI with live SSE streaming) are implemented. Phase 2 (durable workflow with Redis) and Phase 5 (production deploy) are not started.
